# automatic_receipt (会計業務自動化システム)

組織内の経費精算・会計照合業務を自動化・効率化するための Google Apps Script (GAS) システムです。
生成AI（Gemini API）による画像解析と、Gmail APIによるデビットカード利用通知メールの抽出を組み合わせ、突合確認までを完全自動化します。

## 導入効果
* **作業工数の削減**: 毎月のレシート入力および目視による照合時間を **月10時間 → 30分（95%削減）** へ。
* **ミスの撲滅**: 金額や品目の転記ミス、未申請のカード利用の放置を防止。

## システム概要・機能一覧

### 1. レシート画像解析部 (`code.gs`)
* **AIによるマルチモーダル解析**: Googleフォーム等に添付された複数のレシート画像を、Gemini APIを用いて解析。
* **データの構造化**: 解析結果から「日付」「金額」「品目（カテゴリ分類）」を抽出し、スプレッドシートの帳簿形式に自動展開。
* **UI/UXの考慮**: スプレッドシートのカスタムメニューから、特定の行を選択して「再解析」を実行できる手動リカバリ機能を搭載。

### 2. メール抽出＆自動突合部 (`code_mail.gs`)
* **決済メールの自動スクレイピング**: 三菱ＵＦＪ‐ＶＩＳＡビジネスデビットの利用通知メールから、受信日時・利用先・金額・利用者を自動抽出。
* **月次自動集計**: 抽出データから月別の利用金額推移グラフを自動生成。
* **クロスチェック機能**: レシート申請データとメール決済データを「日付×金額」で自動マッチングし、突合が完了した明細のチェックボックスを自動更新。
* **Gmailステータス連携**: チェックが完了したデータに連動し、対応するGmailに「確認済み」ラベルを付与して自動アーカイブ。

## 技術的なこだわり・堅牢性（実務視点）
* **同時実行制御 (排他制御)**
  `LockService` を実装し、複数ユーザーが同時にフォームを送信した際のデータの重複書き込みやデッドロックを完全に防止しています。
* **エラーハンドリング＆自動リトライ機構**
  APIの503エラー（混雑による一時的ダウン）や、Google Driveのファイル取得エラーに対し、ミリ秒単位での段階的な待機（指数バックオフ）を挟んだ**最大3回のリトライ処理**を実装。システムの可用性を高めています。
* **セキュリティ（秘匿情報の分離）**
  APIキーやモデル名などの環境変数はコード内にハードコーディングせず、GASの `ScriptProperties` を利用して安全に管理しています。

## 技術スタック

* Gmail API / Drive API

## 構成・セットアップ（開発者向け）

### スクリプトプロパティの設定
本スクリプトを実行する際は、GASの「プロジェクトの設定」＞「スクリプトプロパティ」に以下の環境変数を設定する。

| プロパティ名 | 説明 |
| --- | --- |
| `GEMINI_API_KEY` | Google AI Studio 等で発行した Gemini API キー |
| `GEMINI_MODEL` | 使用するモデル名（例: `gemini-2.5-flash` など） |
